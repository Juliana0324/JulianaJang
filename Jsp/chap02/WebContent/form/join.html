Form 테그

  • 클라이언트의 입력을 수집하기 위해 사용하는 테그
  • 사용자의 입력을 서버로 전송할 수 있다

    • Form tag속성:

  • action 속성: 사용자의 입력이 전송될 URL을 성정(설정이 없으면 현재 페이지) :: 보낼 페이지
  • method 속성: action에 설정된 URL로 데이터를 보낼때의 HTTP method를 설정한다:: 보낼방식
  • GET 방식- URL에 데이터를 실어보내는 방식(default)
    ex) ?name1=value1&name2=value2&...
    GET방식은 길이 제한이 있어서 너무 긴 데이터를 보내면 데이터 손실 가능성 있음
    사용자가 해당페이지를 즐겨찾기 하기를 원한다면 GET방식을 사용하는 것이 좋다
    ==> GET방식은 주소창에 내용보임
  • POST 방식- request의 body라는 부분에 data를 실어보내는 방식
    ==> 주소창에는 안보임
    POST방식은 길이 제한이없어서 글내용가튼 긴데이터를 보내는 데에 적합
